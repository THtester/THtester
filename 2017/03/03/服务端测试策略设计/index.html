服务端测试策略设计

服务端测试策略设计

冒烟测试阶段

在测试环境搭建完成后，首先要进行冒烟测试
测试点：此阶段主要验证主要的、关键的业务功能是否正常，跑主要测试用例，不用很细，是可以开展测试的一个入口，保证项目的可测性
如果冒烟测试不通过，则打回给开发，待开发再次提交测试

集成测试、系统测试阶段

测试点：此阶段是整个测试的重点阶段，主要保证各个测试类型测试的完整性、全面性、深度，请见下边的各个类型介绍
在众多的测试类型中，测试有先后顺序，一般情况下是先功能、后安全、最后性能

【页面功能测试】
主要保证需求与功能的正确性、准确性、一致性
依据需求文档、前期编写的测试用例，结合前端验证执行
过程中会有首轮测试与多次的回归测试：
第一轮根据不同模块分配不同测试人员，争取较早时间将功能覆盖完毕，bug 上报到 bugzilla
第二轮 bugfix 版本测试，只做 bug 验证，不做其他功能测试，尽快完成 bug 验证
每一轮所需耗费的时间是相应减少的，bug 趋势应该是收敛的，到第三轮、第四轮测试时不应出现严重 bug
在完成多轮的回归测试后，最后要对整体功能做一遍系统测试，主要测试各个功能点是否全部正常

接口功能测试
主要验证传入正常、异常参数，输出结果是否符合预期
依据需求文档、前期编写的测试用例
测试过程与页面功能测试的轮次相同

【UI 测试】
验证页面的布局是否合理
验证页面长度是否合理
验证页面风格是否统一
验证字体大小、颜色是否合适，是否与背景色搭配
验证滚动条的长度是否根据显示信息的长度或宽度能及时变换，以利于用户了解显示信息的位置和百分比

【兼容性测试】
主要考虑容错性：
A、在不同的操作系统上看是否可正常显示图片等信息
B、在不同的分辨率上看是否可正常显示图片等信息
C、在不同的浏览器上的页面显示情况

【安全测试】
待功能测试完成后，进行安全测试（SQL 注入，XSS 注入）
安全测试通过后，即可进行性能测试

【性能测试、稳定性测试】
通过测试工具测试，保证业务系统在高负荷，长时间运转下，响应时间的及时性、稳定性
测试中主要观察点：
数据库、应用服务器、系统内核参数是否已调整
应用处理能力测试：系统最大并发数、系统最优并发数
系统资源占用测试：程序所占系统资源情况
数据库是否分库分表，是否建立索引

测试完成阶段
在测试机上，备份测试通过的版本（用Jenkins管理），记录版本号（commit id）
编写测试总结文档，总结项目结果及测试经验
发送完成确认信给相关人员

上线阶段
主要工作是验证线上功能是否正常

项目上线后，要及时对线上程序进行跟踪验证
若跟踪中发现问题，需要立刻和相关开发人员、产品人员进行反馈、确认
