Android_adb_常用命令

Android adb 常用命令

adb 获取设备系统信息命令

命令 用途
adb root 获取 root 权限
adb -s 6cd08ae0 指定设备 device
adb shell 在 pc 端启动 shell 命令终端建议后面命令放引号内防 grep 报错
adb shell;cd /system/bin;ls 可以列表出大部分可用的命令
adb shell wm size 获取屏幕分辨率
adb shell wm density 屏幕密度
adb shell netstat -alnp 网络统计
adb shell getprop ro.build.version.release 获取 Android 版本号
adb shell getprop ro.product.model 获取设备信息
adb shell getprop ro.product.manufacturer 获取手机品牌信息
adb shell ls -al /data/tombstones 列出目录所有文件
adb shell rm -rf /path/ 删除目录
adb shell rm /path/filename 删除文件
adb monkey -p /path/appname -v 100 随机压力测试
adb shell stop 强行停止某一个服务
adb shell start 启动某一个服务
adb shell top 显示瞬间的进程的资源占用等信息
adb shell free 显示当前进程内存使用情况
adb shell ps 显示进程的自身标识信息
adb shell ps | findstr com.android.browser 获取应用进程信息
adb shell ifconfig | findstr Mask 获取 IP 地址
adb shell cat /proc/cpuinfo CPU 信息
adb shell cat /proc/meminfo 内存信息

adb 查看log日志命令

命令 用途
adb logcat 查看日志
adb logcat -c 清空日志
adb logcat -v time -s TAGNAME 显示自定义的 TAGNAME 并显示时间
adb logcat -b all 获取详细日志
adb logcat > C:\User\admin\Desktop\xxx.log 将日志保存到桌面xxx.log文件中
adb logcat > “%date:0,4%-%date:5,2%-%date:8,2% %time:0,2%时%time:3,2%分%time:6,2%.log” 日期文件名格式保存日志
adb logcat -v format 设置日志的输出格式
adb logcat -v time 显示格式：日期 时间 优先级/标签（进程ID）：进程名词：日志信息
adb logcat -v thread 显示格式：优先级（进程ID：线程ID）标签：日志内容
adb logcat -v brief 默认的日志格式：优先级/标签（进程ID）：日志信息
adb logcat -v process 显示格式：优先级（进程ID）日志信息
adb logcat -v tag 显示格式：优先级/标签：日志信息
adb logcat -v raw 显示格式只输出日志信息，不附加任何其他信息，如：优先级、标签等
adb logcat -v long 显示格式：[日期 时间 进程ID：线程ID 优先级 /标签] 日志信息
adb logcat -t n 显示最近的n行日志
adb logcat *:V 显示 Verbose（明细）的日志
adb logcat *:D 显示 Debug（调试）以上级别的日志
adb logcat *:I 显示 Info（信息）以上级别的日志【默认的日志过滤项】
adb logcat *:W 显示 Warn（警告）以上级别的日志
adb logcat *:E 显示 Error（错误）以及上级别的日志
adb logcat *:F 显示 Fatal（严重错误）以上级别的日志
adb logcat *:S 显示 Silent（Super all output）最高的优先级，可能不会记载东西
adb logcat WifiHW:D *:S 输出10条 标签为 WifiHW，并且优先级 Debug（调试）等级以上级别的日志(没有*:S会报错)
adb logcat WifiHW:D dalikvm:I *:S 输出 标签为 WifiHW，Debug以上级别 和 标签为dalikvm，Info以上级别的日志
adb shell “logcat | grep Wifi” 在日志中过滤出有字符串“Wifi“的日志进行显示
adb shell logcat com.tencent.mm -v time 通过包名来过滤日志
adb logcat |findstr com.tencent.mm 通过包名来过滤日志（如果grep不行的话）

adb 系统操作命令

命令 用途
adb reboot 重启手机
adb shell input keyevent 模拟按键输入
adb shell input keyevent 3 HOME 键
adb shell input keyevent 4 返回键
adb shell input keyevent 164 静音
adb shell input text hello 输入文本
adb shell input swipe 300 1000 300 500 向上滑动手势解锁
adb -P start-server 指定 adb server 网络端口，默认 5037
adb shell settings put global adb_enabled 0 关闭 USB 调试模式
adb shell wm size|density|overscan 修改屏幕属性值
adb shell wm size|density|overscan reset 恢复屏幕属性
adb shell screencap -p /sdcard/sc.png 截图
adb shell screenrecord /sdcard/filename.mp4 录屏，保存为 mp4 格式Ctrl + C停止，默认与最长180秒

adb 应用操作命令

命令 用途
adb install /path/appname 安装应用
-r 允许覆盖安装
-d 允许降级覆盖安装
-g 授予所有运行时权限
-s 将应用安装到 sdcard
-t 允许测试包，允许安装AndroidManifest.xml里application指定android:testOnly=”true”的应用
-H Name of adb server host (default: localhost)
-P Port of adb server (default: 5037)
adb uninstall com.abc.abc 卸载应用
adb push /sourcepath/filename /destinationpath/filename 从 pc 端拷贝一个文件到终端
adb pull /sourcepath/filename /destinationpath/filename 从终端拷贝一个文件到 pc 端
adb shell setprop KEY VALUE 设置 key 的 property 值 value
adb shell getprop KEY 获取该 key 的 property 值

adb 启动 apk 命令

命令 用途
adb shell am start -a android.intent.action.VIEW -d 发送广播消息
am start -n {包名(package)}/{包名}.{活动activity名称} 启动某一个 app 的 activity
adb shell am start -n android/com.android.internal.app.RecommendActivity 拉起应用的 Activity
adb shell am start -n com.kugou.android/.app.splash.SplashActivity 同上
adb shell am startservice -n com.android.clearservice/com.util.job.StartService 启动服务
adb shell am broadcast [options] 发送 指定的广播
adb shell am force-stop com.qk.android.cloudsync 关闭应用
adb shell am start -a android.intent.action.VIEW “snssdk1128://detail?id=6637880297297808653&gd_label=click_schema_xy10” 拉起抖音的deeplink

adb shell dumpsys（查看 DUMP OF SERVICE）

命令 用途
adb shell dumpsys window 查看Window的信息
adb shell dumpsys window displays 显示屏参数
adb shell dumpsys window | findstr mFocusedApp 获取当前窗口打开的 app 信息
adb shell dumpsys activity activities 显示活动栈信息，activities 堆栈信息
adb shell dumpsys package 获取应用详细信息
adb shell dumpsys package com.android.webview 获取系统 webview 信息
adb shell dumpsys meminfo com.tencent.mm 获取指定包名内存使用情况
adb shell dumpsys cpuinfo 查看CPU相关信息
adb shell dumpsys battery 查看电池使用信息
adb shell dumpsys battery -h -h 查看这个service的帮助信息
adb shell dumpsys notification
adb shell dumpsys wallpaper
adb shell dumpsys activity services [] 查看与某包名相关的 Services

adb 查看应用包信息命令

命令 用途
adb shell pm list packages 查看当前连接设备或者虚拟机的所有包
adb shell pm list packages -d 只输出禁用的包
adb shell pm list packages -e 只输出启用的包
adb shell pm list packages -s 只输出系统的包
adb shell pm list packages -i 只输出包和安装信息（安装来源）
adb shell pm list packages -u 只输出包和未安装包信息（安装来源
adb shell pm list packages -f 输出包和包相关联的文件
adb shell pm list packages -3 输出所有第三方包
adb shell pm list packages -[option] “sina” 按照要求搜索包
adb shell pm clear 清除应用数据与缓存

adb 刷机相关命令

命令 用途
adb reboot recovery 从 Recovery 重启到 Android
adb reboot bootloader 重启到 Fastboot 模式
adb sideload <path-to-update.zip> 通过 adb 上传和更新系统

aapt

命令 用途
aapt dump badging /apks/abc.apk 获取 apk 所有信息

Android adb 的使用

Android adb 简介

adb 全称 Android Debug Bridge 用来操作Android设备，起到调试桥的作用

adb的作用：借助adb工具，可以管理设备的状态，对设备进行安装软件、系统升级、运行shell命令等

adb 下载：https://developer.android.google.cn/studio （下载后配置好环境变量）

Android adb的使用1 — 通过数据线连接手机

window系统：win+R 键 输入cmd 命令进入命令窗口，通过数据线把测试设备连接到电脑

Mac系统：打开终端，通过数据线把测试设备链接到电脑

冒烟测试阶段

在测试环境搭建完成后，首先要进行冒烟测试
测试点：此阶段主要验证主要的、关键的业务功能是否正常，跑主要测试用例，不用很细，是可以开展测试的一个入口，保证项目的可测性
如果冒烟测试不通过，则打回给开发，待开发再次提交测试

集成测试、系统测试阶段

测试点：此阶段是整个测试的重点阶段，主要保证各个测试类型测试的完整性、全面性、深度，请见下边的各个类型介绍
