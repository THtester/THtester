Android adb 的使用

Android adb 简介

adb 全称 Android Debug Bridge 用来操作Android设备，起到调试桥的作用

adb的作用：借助adb工具，可以管理设备的状态，对设备进行安装软件、系统升级、运行shell命令等

adb 下载：https://developer.android.google.cn/studio （下载后配置好环境变量）

Android adb的使用1 — 通过数据线连接手机

window系统：win+R 键 输入cmd 命令进入命令窗口，通过数据线把测试设备连接到电脑

Mac系统：打开终端，通过数据线把测试设备链接到电脑

连接出现device not found

电脑连接部分 Android 手机会出现 adb devices 返回 device not found，确认电脑 adb 环境是 ok 的，那么需要检查以下几项：

1、手机连接电脑时，选择 USB 连接方式：文件传输

2、开发人员选项 – USB调试模式，打开

3、手机连接电脑时，手机上提示是否信任此电脑，勾选并确认

4、确认下，数据线是否支持数据传输，还是只是充电线

在以上检查都通过后，还是 device not found，可以通过绑定 adb 的硬件 VID 解决问题：

Windows 方案：

1、计算机–属性–设备管理器–便携设备–右键属性–详细信息–硬件 Id，找到如：VID_12D1

2、C:\Users\admin.android 文件夹下，新建 adb_usb.ini

3、在 adb_usb.ini 中写入：0x12D1

Mac 方案：

1、执行 system_profiler SPUSBDataType 获得 Vendor ID：0x0bda

2、/Users/admin/.android 文件夹下，新建 adb_usb.ini

3、在 adb_usb.ini 中写入：0x0bda

重启 adb，然后执行 adb devices 就可以获取到设备信息了：

Android adb的使用2 — 通过WiFi连接手机

1、先用 USB 连接手机，让设备在 5555 端口监听 TCP/IP

2、断开 USB，在「设置」-「关于手机」-「状态信息」-「IP地址」找到手机 IP 地址

如：192.168.1.16

3、通过 IP 地址连接设备

4、断开连接
