Centos7 安装部署 HttpRunnerManager

Centos7 安装部署 HttpRunnerManager V2.0

HttpRunnerManager V2.0此项目是一个基于HttpRunner的接口自动化测试平台，同时HttpRunner 是一款面向 HTTP(S) 协议的通用测试框架

只需编写维护一份 YAML/JSON 脚本，即可实现自动化测试、性能测试、线上监控、持续集成等多种测试任务

一、准备环境和安装包

主要：CentOS7，HttpRunnerManager V2.0，mysql（MariaDB），python3.7，Django2.1.2，rabbitmq-server-3.7.15

系统下载地址：

CentOS：https://mirrors.edge.kernel.org/centos/7.6.1810/isos/x86_64/CentOS-7-x86_64-DVD-1810.iso

HttpRunnerManager V2.0：https://github.com/HttpRunner/HttpRunnerManager

mysql，python，rabbitmq （安装略）

在 mysql 中新建 HttpRunner 和 HttpRunner_debug（用于DEBUG版本）数据库，charset utf8

二、修改项目设置

1、打开下载后解压的源码包，进入 HttpRunnerManager/HttpRunnerManager 文件夹下，找到 settings.py 打开修改

$ vim settings.py

（1）DEBUG = False

（2）DATABASES 字段，填写使用的数据库信息（区分 DEBUG）

（3）修改邮箱配置：EMAIL_SEND_USERNAME和EMAIL_SEND_PASSWORD（要打开 smtp 服务）

（4）修改 djcelery.setup_loader() 下 rabbitmq-server 相关配置 ：BROKER_URL 和 CELERYD_MAX_TASKS_PER_CHILD = 200

2、mysql 文件 mysql_config 拷贝到 /usr/local/下
ln -s /usr/bin/mysql/bin/mysql_config /usr/local/bin/mysql_config

3、安装 python 依赖模块（也可以 pip3 install -r requirements.txt，但是提供的版本不准确）

三、运行

1、数据库 mysql 服务和 RabbitMQ 服务处于启动状态

2、执行 migrate 和创建管理员

3、启动服务

四、登录管理

管理员登录页面：http://127.0.0.1:9003/admin/

用户登录页面：http://127.0.0.1:9003/api/login/

五、异步任务和 \dashboard****

1、rabbitmq 开启 web 插件

测试访问：http://127.0.0.1:15672/ 默认账号密码：guest guest（这个账号只允许本机访问）

2、修改 templates/base.html

将 http://192.168.91.45:5555/dashboard 改为 http://127.0.0.1:5555/dashboard

3、修改 HttpRunnerManager/settings.py

BROKER_URL = ‘amqp://dev:zwc123@192.168.91.45:5672//‘ if DEBUG else ‘amqp://dev:zwc123@192.168.91.45:5672//‘

改为

BROKER_URL = ‘amqp://guest:guest@localhost:5672//‘ if DEBUG else ‘amqp://guest:guest@localhost:5672//‘
