简单回顾下 Monkey

1、Android Monkey

基本命令：

monkey -p com.baidu.searchbox –monitor-native-crashes –ignore-timeouts –kill-process-after-error –pct-touch 50 –pct-motion 30 –pct-appswitch 20 -s 100 -v -v –throttle 200 100 > /sdcard/tmp/monkey_sdk.log

2、tv.panda.test.monkey

基本命令：

adb shell CLASSPATH=/sdcard/monkey.jar:/sdcard/framework.jar exec app_process /system/bin tv.panda.test.monkey.Monkey -p com.baidu.searchbox –uiautomatormix –running-minutes 180 -v –throttle 100 >./monkeyout.txt 2>./monkeyerr.txt &

Monkeyrunner 同 Monkey 的区别

Monkey 工具直接运行在 adb shell 中，生成用户或系统的伪随机事件流

Monkeyrunner 工具则是在工作站上通过 API 定义的特定命令和事件控制设备或模拟器

Monkeyrunner 支持自己编写插件、控制事件、随时截图、记录和回放

MonkeyRunner APIs（Monkeyruner 脚本为 python 语言编写）

MonkeyRunner 用来连接设备或者模拟器

MonkeyDevice 提供安装/卸载应用, 发送模拟事件

MonkeyImage 完成图像保存及对比的操作

1、MonkeyRunner

alert 警告框

waitForConnection 等待设备连接，有多个 device, 需要指明具体 device id

sleep 添加等待时间

2、MonkeyDevice

drag 拖动（起点位置, 终点位置, 手势持续的时间, 插值点的步数默认10）

press 按键

startActivity 启动应用

touch 点击（x坐标，y坐标）type: DOWN, UP, DOWN_AND_UP

type 输入

takeSnapshot 截屏

3、MonkeyImage

sameAs 图像对比

writetoFile 保存图像文件，path: 文件存储的路径，format: 图像文件的格式png ,JPG等

Monkeyrunner 的测试类型

1、多设备控制

Monkeyrunner API 可以跨多个设备或模拟器实施测试套件

2、功能测试

Monkeyrunner 可以为一个应用自动执行一次功能测试（提供按键或触摸事件的输入数值，然后观察输出结果的截屏）

3、回归测试

Monkeyrunner 可以运行某个应用，并将其结果截屏与既定已知正确的结果截屏相比较，以此测试应用的稳定性

4、可扩展的自动化

由于 Monkeyrunner 是一个 API 工具包，可以基于 Python 模块和程序开发一整套系统，以此来控制 Android 设备

与monkey对比优势：

快

稳：只进行有意义的操作，防误点状态栏，不会乱断网、卸载应用

支持脱机运行，错误日志可以记录到手机本地

支持自定义：控件黑白名单，自定义操作步骤，控件操作优先级

参数含义：

-p com.test.test：指定要运行的包名，需要替换成你在测试的应用

–uiautomatormix：遍历方式，其它两个不稳定，可以默认用这个

–running-minutes 2880：指定运行时间，单位是分钟，2880就是2天

-v：log输出级别

–throttle 500：每个动作之间间隔，如果不加会以尽可能快的速度发送指令。为了模拟正常操作，一般设置在300以上

2>/sdcard/monkeyerr.txt：如果运行过程中出现崩溃卡死，错误文件会保存在这里。
